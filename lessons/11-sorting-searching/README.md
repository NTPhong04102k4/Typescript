# 11 · Sorting & Searching

Sorting and searching are the backbone of algorithmic thinking: most
interview problems either need data in order or need to find something
fast once it is. This topic walks from the simple O(n^2) comparison sorts
up through divide-and-conquer (merge sort, quicksort), non-comparison
sorts that beat the O(n log n) lower bound (counting/radix), and then a
deep dive into binary search and the "search on the answer" pattern that
shows up constantly in interviews. It closes with a pattern lesson and a
practice set that ties everything together.

Lessons:
- 01 — Comparison sorts overview (bubble/insertion/selection), ASCII trace
- 02 — Merge sort & quicksort (divide and conquer)
- 03 — Non-comparison sorts (counting/radix) & engine array notes
- 04 — Binary search deep dive & search-space reduction
- 05 — Sorting/searching interview patterns
- 06 — LeetCode practice set: Sorting/Searching
